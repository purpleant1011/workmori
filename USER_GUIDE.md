## Workmori란?

Workmori는 사장님 가게의 **SNS 홍보·고객 응대·콘텐츠 제작·마케팅 일정**을 도와주는 **가상 AI 직원**입니다.

핵심 특징:
- **기억하는 AI 직원** — 가게의 상품, 메뉴, 자주 받는 질문, 톤, 응대 원칙을 모두 학습합니다.
- **채널 자동 게시** — 인스타그램, 카카오 채널, 네이버 플레이스 등 채널에 자동 게시합니다.
- **24시간 고객 응대** — AI 직원이 첫 응대를 하고, 어려운 문의만 사장님께 넘깁니다.
- **주간 자동 일정** — 매주 콘텐츠/홍보/응대 일정을 AI가 짜고 실행합니다.
- **검수 모드 → 자동 모드** — 처음엔 사람이 검수하고, 안심되면 자동으로 실행합니다.

> ⚠️ **약속과 주의사항**
> - Workmori는 **마케팅 결과를 보장하지 않습니다** (조회수·매출 등).
> - 과대 광고·허위 후기·전후사진 100% 같은 표현은 **자동 차단**됩니다.
> - 의료·법률·금융 등 민감 문의는 AI가 답하지 않고 사람 상담으로 안내합니다.
> - 실제 고객 응대 전에는 **테스트 모드**로 충분히 검증하시기 바랍니다.

## 첫 시작: 가입과 로그인

### 가입
1. `https://workmori.example/users/sign_up` 접속
2. 이메일과 비밀번호 입력 → 인증 메일 확인 → 가입 완료
3. 첫 화면에서 **사업장 정보** 입력

### 데모 로그인 (개발 환경)
```
http://localhost:3001/dev_login/business
이메일: owner@demo.example
```
→ 데모 계정으로 즉시 로그인됩니다.

## 1단계: 사업자 프로필 입력

> 메뉴: **사업장** → **프로필 편집**

사장님의 가게가 **어떤 곳인지**, **누구를 위한 곳인지**, **어떤 톤으로 말하는지** 알려주세요. AI 직원이 이 정보를 기본으로 학습합니다.

### 기본 정보
- 상호(가게 이름)
- 업종 (예: 한식 음식점, 미용실, 온라인 쇼핑몰)
- 주소·연락처·영업시간
- 대표자명, 사업자등록번호 (선택)
- 가게 소개 (2~3줄로 자유롭게)

### 타깃 고객
- 주 고객층 (예: 30대 직장인 여성, 가족 단위)
- 자주 오는 지역 (동네, 상권)
- 가격대 (저가/중가/고가)

### 브랜드 톤
- 친근함 / 정중함 / 전문적 / 위트 있는 / 감성적
- 금지 표현 (예: "100% 효과", "완치")
- 자주 쓰는 인사말, 마무리 멘트

> 💡 **Tip**: 톤은 구체적일수록 좋습니다. "말 편하게"보다는 "반말 기본, 이모티콘 1~2개"처럼 적어주세요.

## 2단계: 지식 베이스 만들기

> 메뉴: **지식**

AI 직원이 모르는 것을 알려주는 단계입니다. **5가지 형태**로 등록 가능합니다.

### 1) 상품 (Products)
- 메뉴판, 상품 카탈로그
- 가격, 옵션, 재료, 알레르기 정보
- 사진, 추천搭配

### 2) 서비스 (Services)
- 시술/서비스 목록
- 소요 시간, 가격, 예약 방법
- 전후 주의사항

### 3) 자주 묻는 질문 (FAQ)
- 영업시간, 주차, 예약, 환불 등
- 질문 + 답변 형태

### 4) 문서 (Knowledge Sources / Knowledge Documents)
- PDF 메뉴판, 회사 소개서, 안내문
- 파일을 업로드하면 자동으로 텍스트 추출·색인
- 검색하면 관련 문서를 찾아 인용

### 5) 톤·정책 메모 (자유 메모)
- "절대 '무료'라고 말하지 마세요"
- "신메뉴 출시 시 'NEW' 이모지 사용"
- 브랜드 가이드라인

> 💡 **Tip**: 처음에는 **상품 5개 + FAQ 10개 + 자유 메모 3개**만 등록해도 충분합니다. AI가 부족한 부분을 표시해주면 그때 추가하세요.

## 3단계: AI 직원 만들기

> 메뉴: **사업장** → **AI 직원 추가**

사업장마다 여러 AI 직원을 둘 수 있습니다. 예:
- **홍보 담당** (콘텐츠 발행 전담)
- **CS 담당** (고객 응대 전담)
- **예약 담당** (예약·변경·취소 전담)

### AI 직원 설정 항목

| 항목 | 설명 |
|------|------|
| 이름 | 예: "모리", "루나", "카리스" |
| 역할 | 위 역할 중 선택 |
| 말투 | 친근/정중/위트 등 |
| 사용 가능 채널 | 인스타/카카오/이메일 등 |
| 검수 모드 | on/off |
| 응대 가능 시간 | 09~22시 등 |
| 답변 길이 | 짧게(1~2줄) / 보통(3~5줄) / 자세히 |
| 응대 불가 주제 | 의료/법률/금융 등 민감 주제 |
| 에스컬레이션 규칙 | "분노 고객은 사람에게", "결제 분쟁은 사람에게" |

## 4단계: 채널 연결

> 메뉴: **채널**

연결할 수 있는 채널:

| 채널 | 용도 | 인증 |
|------|------|------|
| Instagram | 피드/릴스/스토리 자동 게시 | OAuth |
| 카카오 채널 | 친구 응대, 메시지 발송 | 카카오 비즈니스 인증 |
| 네이버 플레이스 | 리뷰 응대, 정보 업데이트 | 네이버 비즈니스 로그인 |
| Mastodon | 텍스트 중심 SNS | 액세스 토큰 |
| 이메일 | 고객 문의, 뉴스레터 | SMTP/IMAP |
| 웹 위젯 | 자체 사이트 채팅 | embed script |

> 💡 **Tip**: 처음엔 **Instagram + 카카오 채널** 두 개만 연결하고, 운영 안정화 후 추가하세요.

## 5단계: 주간 일정 짜기

> 메뉴: **일정**

매주 AI 직원이 **자동으로 실행할 작업의 타임라인**을 정의합니다.

### 기본 템플릿

| 요일 | 오전 | 오후 | 저녁 |
|------|------|------|------|
| 월 | 신규 콘텐츠 1건 발행 | FAQ 자동 학습 | 미응대 답변 응대 |
| 화 | 상품 하이라이트 릴스 | 댓글 응대 | - |
| 수 | - | 블로그 글 발행 | - |
| 목 | 신규 콘텐츠 1건 발행 | FAQ 자동 학습 | - |
| 금 | 주간 리포트 작성 | 고객 응대 리뷰 | - |
| 토 | 시즌 프로모션 | 응대 | 응대 |
| 일 | 휴식 | 휴식 | 휴식 |

각 작업에는 **검수 필요 여부**를 정할 수 있습니다.
- **검수**: 사람이 보고 [승인] 해야 게시
- **자동**: 즉시 게시 (충분히 안심된 후에만 켜기)

## 6단계: 첫 콘텐츠 만들기

> 메뉴: **콘텐츠** → **새 콘텐츠**

콘텐츠 유형:
- **Instagram 피드** (이미지 + 캡션)
- **Instagram 릴스** (숏폼)
- **블로그** (장문)
- **카카오 채널 메시지**
- **이메일 뉴스레터**

### 작성 흐름
1. **주제** 입력 (예: "여름 신메뉴 콜드브루")
2. AI가 초안 생성
3. 사장님이 수정·승인
4. 발행 채널 선택
5. 예약 발행 or 즉시 발행

> 💡 **Tip**: 처음에는 검수 모드 **on**으로 시작해서, AI가 만드는 초안이 마음에 들면 점점 자동 비율을 높이세요.

## 7단계: 검수과 자동 실행

> 메뉴: **콘텐츠** → **검수 큐**

AI가 만든 콘텐츠는 검수 큐에 쌓입니다. 사장님이 직접 보고 [승인/거절/수정] 합니다.

- **승인** → 즉시/예약 발행
- **거절** → 폐기 (사유 기록)
- **수정** → 다시 작성 또는 직접 편집

### 자동 검수 기준 (AI가 자체 검사)
- 과대 광고 표현 (예: "100%", "완치")
- 금지어
- 진술 가능한 사실관계
- 톤 일관성

⚠️ 자동 검수를 통과해도 **반드시 사람이 한 번 더 보시기 바랍니다** (의료·법률·금융 제외).

## 8단계: 대화 응대하기

> 메뉴: **대화**

### 모드
- **AI 자동 응대**: 모든 첫 응대 AI가 처리
- **검수 후 응대**: AI 초안 → 사람 검수 → 발송
- **사람 직접 응대**: 모든 응대 사람이 (필요 시 AI 보조)

### 에스컬레이션 (사람에게 넘기는 조건)
- 부정 감정 강한 고객
- 결제·환불 분쟁
- 민감 주제 (의료·법률·금융)
- AI 확신도가 낮은 질문

### CSAT (만족도)
- 매 대화 끝나면 고객에게 1~5점 평가 요청
- 평균 4.0 미만이면 자동 알림 → 사장님이 직접 확인 권장

## 9단계: 결과 확인과 리포트

> 메뉴: **리포트**

### 주간 리포트 (매주 금요일 자동 생성)
- 발행된 콘텐츠 수, 채널별 도달
- 대화 수, 평균 응대 시간
- CSAT 평균, 에스컬레이션 수
- 자동화 규칙 발동 수

### 월간 리포트
- 채널별 팔로워 변화
- 콘텐츠별 인게이지먼트
- 매출 기여 추정 (UTM 기반)

## 10단계: 데이터 백업받기

> 메뉴: **백업** (`/app/data_exports`)

사장님의 모든 데이터를 직접 다운로드할 수 있습니다.

### 내보내기 종류 (9종)
- **minimal**: 가게 정보 + AI 직원 기본
- **conversations**: 모든 대화 + 메시지
- **content**: 콘텐츠 + 발행 이력 + 만족도
- **products**: 상품 + 서비스
- **knowledge**: 지식 베이스 (FAQ/문서)
- **reports**: 리포트 + 감사 로그 + 만족도
- **billing**: 구독 + 청구 + 결제 + 계약
- **audit**: 감사 로그 + 발송 로그
- **full**: 전체

### 형식 (3종)
- **JSON**: 단일 파일, 구조 보존
- **CSV** (zip): 테이블별 분리된 CSV 묶음
- **ZIP**: data.json + csv/ 폴더

### 파일 위치
- 다운로드: 해당 페이지의 [다운로드] 버튼
- 저장 위치: `storage/exports/account_<id>/`
- 보관 기간: **30일** 후 자동 삭제 (보관 기간 변경 가능)

## 14단계: 자동화 규칙

> 메뉴: **설정** → **자동화**

반복되는 작업을 AI가 알아서 처리하도록 규칙을 만듭니다.

### 예시 규칙
- "리뷰 평점 5점이면 자동으로 답글 달기"
- "DM으로 '예약' 키워드 오면 예약 폼 링크 보내기"
- "주말에 부정 댓글 오면 사장님께 알림"
- "매일 오전 10시에 신메뉴 1개 추천 포스팅"

### 규칙 구조
```
WHEN [트리거] (예: 새 DM 도착, 새 댓글, 매일 HH:MM)
IF  [조건]   (예: 평점 >= 4, 키워드 포함, 고객이 VIP)
DO  [액션]   (예: 답글 생성, 알림 발송, 콘텐츠 발행)
```

## 요금제와 결제

> 메뉴: **결제**

| 요금제 | 월 요금 | 포함 |
|--------|---------|------|
| Starter | 11만원 | AI 직원 1명, 채널 1개, 콘텐츠 8건/월 |
