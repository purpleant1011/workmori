# 제품 요구사항 (PRD) — 워크모리(가칭)

> 사장님의 일을 기억하고 스스로 이어가는 AI 직원 — 소상공인 마케팅 직원을 첫 번째로.

## 1. 사용자

| 역할 | 범위 |
|---|---|
| 플랫폼 최고 관리자 | 모든 사업자, 계약, 기능 플래그, 장애, 작업, 비용, 보안 사건 |
| 플랫폼 운영자 | 고객 대신 AI 직원 설정, 자료 정리, 게시 승인 지원 |
| 사업자 소유자 | 자기 사업과 AI 직원 전체 |
| 사업자 관리자 | 소유자가 허용한 기능 |
| 사업자 검토자 | 콘텐츠 승인/반려만 |
| Hermes 실행 주체 | API 호출, 사람 화면 없음 |

## 2. 핵심 흐름

```
[1] 사업자 가입 · 이메일 인증
[2] 사업자 정보 입력 (브랜드, 지역, 영업시간, FAQ, 금지어, 사람연결 규칙)
[3] AI 직원 설정 (이름, 말투, 비용 한도, 승인 방식, 시간/요일, 자연어 지시)
[4] 자료 업로드 (PDF/텍스트/URL/FAQ/상품) → 처리 → RAG 지식화
[5] 주간 일정 / 자연어 자동화 → 초안 → 사용자가 활성화
[6] 자동화 실행 (Hermes) → 초안 작성 → 검수 → 승인 → 게시
[7] Discord 대화 수신 → RAG 답변 초안 → 위험 문의는 사람 연결
[8] 결과 기록 → 비용 누적 → 보고서
[9] 해지 시 채널 회수 + 자료 내보내기/삭제
```

## 3. 첫 고객: 바이름 청라점

- 미용 분야 첫 파트너
- 테스트 계정에서 품질 검증 → 공식 채널 전환
- 정식 서비스 시작 전까지는 월 이용료 없음
- 공식 시작일이 월 결제 기준일
- 바이름 전용 가격: **월 300,000원 (공급가액) + 부가세 별도, 보증금 500,000원** — 다른 가입자에게 비공개
- 운영자가 대신 사업자 프로필·AI 직원을 입력 (`operator_managed`)
- **공식 인스타/블로그/당근/플레이스에는 승인된 콘텐츠만 게시**

## 4. 자동화 안전성

- `draft → ready → queued → claimed → running → awaiting_approval → approved → publishing → succeeded`
- 실패/중단: `retry_scheduled | failed | cancelled | paused | quarantined | expired`
- 멱등키, 계정/채널 동시성 제한, 지수형 재시도, 일일 비용 상한
- 전체/계정/직원 중단 스위치
- 자연어 지시는 구조화된 초안 → 사용자 확인 → 활성화
- 실행/게시 직전 정책 재검사

## 5. AI 안전성

- 시술 가능 여부, 잔흔, 피부 상태, 클레임, 가격, 환불, 민감 사정 → AI가 임의로 확정하지 않고 사람 연결
- 100% 안전, 부작용 없음, 무조건 자연, 완벽 보장 등 단정 표현 차단
- 상위노출/예약/매출 보장, 가짜 후기, 리뷰 조작, 과장 광고 금지
- 고객 사진/상담/개인정보는 AI 학습 사용 안 함

## 6. 다중 고객 보안

- 모든 업무 데이터 `account_id` 필수
- 기본 거부 권한 모델
- 다른 사업자 데이터 격리 자동 테스트
- RLS는 차기 단계, MVP는 애플리케이션 스코프

## 7. 성공 기준

`docs/testing.md` 참조. 핵심:
