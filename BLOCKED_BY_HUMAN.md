# 사람 작업 필요 (Blocked by Human)

본 문서는 외부에서만 해결할 수 있고, 코드로 우회할 수 없는 항목을 모은다. 각 항목은 누가 무엇을 해야 하는지 명시한다.

| ID | 주제 | 필요한 사람/조직 | 이유 | 대체 동작 |
|---|---|---|---|---|
| B-01 | 도메인 구매 | 퍼플앤트 운영팀 | 가칭 워크모리 도메인 확정 | 화면/문서 모두 "(가칭)" 표시 |
| B-02 | 상표 등록 | 법무/특허팀 | 워크모리/WORKMORI 상표 가능성 | 가칭 표기 유지 |
| B-03 | 사업자등록 | 퍼플앤트 운영팀 | 통신판매업/부가가치세 | "수동 결제" 모드로 시작 |
| B-04 | PG 계약 | 퍼플앤트 운영팀 | 실제 결제사(Stripe/토스페이먼츠) | `ManualBillingProvider` 자리표시 |
| B-05 | Discord 봇 토큰 | 바이름(또는 첫 고객) | 디스코드 디벨로퍼 포털 발급 | `FakeDiscordAdapter` 로 검증 |
| B-06 | 인스타/블로그/당근 OAuth | 각 플랫폼 | 채널별 디벨로퍼 등록 | `ChannelConnection.status = planned` |
| B-07 | 바이름 공식 채널 자격 | 바이름 원장 | 자격 증명 전달 | `access_status = not_received` 기본 |
| B-08 | 이용약관 / 개인정보 처리방침 | 법무 검토 | 법적 효력 | draft 텍스트로 placeholder |
| B-09 | 보험/배상 책임 | 법무 | 자동화 게시 오류 책임 | "가짜 후기/리뷰 조작 금지" 정책 명시 |
| B-10 | 환불 정책 적용 | 법무 | 전자상거래법 | draft 만 작성, 실제 적용 전 법무 검토 |
| B-11 | 다중 인증(MFA) 적용 | 보안 운영 | 실제 운영 전 필수 | UI 인터페이스만 노출 |
| B-12 | 카카오/네이버 스마트스토어 연동 | 각 플랫폼 | 공식 API 등록 | `planned` 상태 |
| B-13 | 첫 고객(바이름) 운영 데이터 입력 | 퍼플앤트 운영팀 이호철 | 사업자/브랜드/금지어/FAQ | `operator_managed=true` 시 운영자가 직접 입력 |
| B-14 | OpenAI/Anthropic/Minimax API 키 | 퍼플앤트 운영팀 | 비용 발생 | `stub` 모드로 자리표시자 |
| B-15 | pgvector 확장 설치 | 인프라 | 벡터 검색 | 미설치 시 키워드 가중치로 폴백 |

위 항목 중 어느 것도 MVP 자체의 동작을 막지 않는다. 각 항목은 자리표시자 / fake adapter / 운영자 직접 입력으로 우회되며, 운영 전 채워 넣는다.
